KMS - Restrict IAM Inline Policies from Decrypt All KMS Keys

Policy Type: tfpolicy

Detailed Description:
Convert following Sentinel Policy to Terraform Policy

```sentinel
# This policy checks if resources of type 'aws_iam_policy_document'
# contain blocked action patterns such as 'kms:ReEncryptFrom' and 'kms:Decrypt'

# Copyright IBM Corp. 2024, 2025
# SPDX-License-Identifier: BUSL-1.1

import "tfstate/v2" as tfstate
import "tfresources" as tf
import "report" as report
import "collection" as collection
import "collection/maps" as maps

# Constants

const = {
	"policy_name":                      "kms-restrict-iam-inline-policies-decrypt-all-kms-keys",
	"message":                          "Actions 'kms:ReEncryptFrom' and 'kms:Decrypt' must not be allowed on all 'KMS keys'. Refer to https://docs.aws.amazon.com/securityhub/latest/userguide/kms-controls.html#kms-2 for more details.",
	"resource_aws_iam_policy_document": "aws_iam_policy_document",
}

# Functions

get_violations = func(resources) {
	return collection.reject(resources, func(res) {
		statements = maps.get(res, "values.statement", [])
		for statements as statement {
			actions = maps.get(statement, "actions", [])
			if actions contains "kms:ReEncryptFrom" or actions contains "kms:Decrypt" {
				return false
			}
		}
		return true
	})
}

# Variables

policy_document = tf.state(tfstate.resources).type(const.resource_aws_iam_policy_document).resources
violations = get_violations(policy_document)

summary = {
	"policy_name": const.policy_name,
	"violations": map violations as _, v {
		{
			"address":        v.address,
			"module_address": v.module_address,
			"message":        const.message,
		}
	},
}

# Outputs

print(report.generate_policy_report(summary))

# Rules

main = rule {
	violations is empty
}
```

Related Terraform Resources:

1. Resource Type: aws_iam_policy_document (Data Source)
   Description: Generates an IAM policy document in JSON format for use with resources that expect policy documents
   Key Arguments:
   - statement: (Optional) Configuration block for a policy statement
     - actions: (Optional) List of actions that this statement either allows or denies (e.g., ["kms:Decrypt", "kms:ReEncryptFrom"])
     - resources: (Optional) List of resource ARNs that this statement applies to (e.g., ["*"] for all resources)
     - effect: (Optional) Whether this statement allows or denies the given actions. Valid values are "Allow" and "Deny". Defaults to "Allow"
   - version: (Optional) IAM policy document version. Defaults to "2012-10-17"
   Attributes:
   - json: Standard JSON policy document rendered based on the arguments

2. Resource Type: aws_iam_role_policy
   Description: Provides an IAM role inline policy
   Key Arguments:
   - name: (Optional) The name of the role policy
   - role: (Required) The name of the IAM role to attach the policy to
   - policy: (Required) The inline policy document. This is a JSON formatted string. Can use jsonencode() or reference aws_iam_policy_document data source
   Note: This resource is for inline policies attached directly to IAM roles

3. Resource Type: aws_iam_user_policy
   Description: Provides an IAM policy attached to a user (inline policy)
   Key Arguments:
   - name: (Optional) The name of the policy
   - user: (Required) IAM user to which to attach this policy
   - policy: (Required) The policy document. This is a JSON formatted string. Can use jsonencode() or reference aws_iam_policy_document data source
   Note: This resource is for inline policies attached directly to IAM users

4. Resource Type: aws_iam_group_policy
   Description: Provides an IAM policy attached to a group (inline policy)
   Key Arguments:
   - name: (Optional) The name of the policy
   - group: (Required) The IAM group to attach the policy to
   - policy: (Required) The policy document. This is a JSON formatted string. Can use jsonencode() or reference aws_iam_policy_document data source
   Note: This resource is for inline policies attached directly to IAM groups

Clarifications:
- The original Sentinel policy only checks aws_iam_policy_document data sources, but inline policies in Terraform are actually created using resources like aws_iam_role_policy, aws_iam_user_policy, and aws_iam_group_policy
- The Terraform Policy should check these inline policy resources and their associated policy documents to ensure they don't allow kms:ReEncryptFrom or kms:Decrypt actions on all KMS keys
- The policy should specifically flag violations when these actions are allowed with "Resource": "*" (all KMS keys), as per AWS Security Hub control KMS.2
- The policy document can be specified either inline using jsonencode() or by referencing an aws_iam_policy_document data source
